# Skill: Jules Orchestrator

## Purpose
To act as the quality gate and release manager for code generated by Jules, ensuring all contributions follow the project's strict `AGENTS.md` standards and maintain a clean git history.

## Workflow

### 1. Discovery
- Run `jules remote list --session` to find sessions associated with the current repository.
- Identify sessions that are in `Awaiting Review`, `Completed`, or have available diffs.

### 2. Ingestion & Sanitization
- For each identified session:
  - Run `jules remote pull --session <ID> --apply`.
  - Scan all modified files for emojis (e.g., 🧪, 🧹, 🔒) or bot-style comments and remove them.
  - Verify that the changes do not violate any core rules in `AGENTS.md`.

### 3. Verification
- Run the test suite: `ddev composer test`.
- Ensure no regressions were introduced by the Jules code.

### 4. Professional Commit
- Prepare a commit message following the **Conventional Commits** specification.
- Adhere to the strict mapping in `AGENTS.md`:
  - `add` -> `feat:`
  - `fix` -> `fix:`
  - `remove/update/refactor` -> `refactor:`
- Propose the commit message to the user for final approval.

### 5. Reporting
- Before committing, provide a detailed report of all discovered sessions in the following format:
  ```text
  X pending tasks found:
  ID [Title/Focus] - [Status: Integrated/Redundant]
  ...
  
  Ready to commit in Y commits with messages:
  
  [Commit Message 1]
  - [File A]
  - [File B]
  ...
  ```

### 6. Finalization
- Once approved, commit and push to master (or merge the corresponding PR using `gh pr merge --squash`).
- Archive the session in Jules if possible (or notify the user to do so).
